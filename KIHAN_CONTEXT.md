## HOMEPAGE STRUCTURE
1. Announcement bar (thin, one line — offer or mood)
2. Full screen hero + one bold line (sells a feeling, not a product)
3. Brand moment — full width image + one line of copy, no CTA (stillness)
4. Campaign blocks — 2 curated collections, editorial style
5. Featured products — 4 max, lifestyle photography, not catalogue grid
6. Journal / Stories teaser — invitation, not content dump
7. Clean minimal footer

## BRAND VOICE
Feel: The pause in today's fast-paced life. Pampering oneself with calm, aesthetic, luxury.
Tone: Soft but confident. Never loud. Speaks like a friend who has figured out how to slow down.
Copy style: Short lines. Speaks to how something feels, not what it is.
References: framesbyhan (intimate framing, personal), Mejuri (accessible luxury), Marc Jacobs (bold editorial)
NOT: eco-conscious messaging, preachy, performative, overly minimal to the point of cold

## DECISIONS MADE
- Supabase for PostgreSQL database (free tier)
- Railway for Medusa backend hosting
- Vercel for Next.js frontend (free tier)
- Razorpay for India-first payments
- Two separate admin dashboards
- Homepage modelled on Mejuri feeling + Marc Jacobs editorial style
- create-medusa-app CLI skipped (interactive-only, can't be automated) — project scaffolded manually
- @medusajs/payment-manual and @medusajs/fulfillment-manual do NOT exist as separate packages in v2 — bundled inside @medusajs/medusa
- Supabase connection must use Session mode port 5432 (NOT transaction pooler port 6543) — Medusa migrations need persistent connections
- File storage: @medusajs/file-local for dev; will switch to S3/Cloudflare R2 for Railway production
- Razorpay: no official Medusa v2 module exists — will need custom payment provider implementation in Phase 4
